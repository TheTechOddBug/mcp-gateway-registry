## Upgrading from 1.27.1

This section covers everything you need to know to upgrade from 1.27.1 to 1.28.0.

### Breaking Changes

There are no breaking changes in this release.

### New Environment Variables

No new environment variables in this release.

### Upgrade Instructions

#### Docker Compose

```bash
cd mcp-gateway-registry
git pull origin main
git checkout 1.28.0

# Review new env vars in .env.example and update your .env if needed
# Then rebuild and restart:
./build_and_run.sh
```

#### Kubernetes / Helm (EKS)

```bash
cd mcp-gateway-registry
git pull origin main
git checkout 1.28.0

# Update values.yaml if needed, then upgrade:
helm upgrade mcp-gateway charts/mcp-gateway-registry-stack -f your-values.yaml
```

No files under `charts/` changed between 1.27.1 and 1.28.0, so a `helm dependency build` / `helm dependency update` is not required for this upgrade.

#### Terraform / ECS

```bash
cd mcp-gateway-registry
git pull origin main
git checkout 1.28.0

# Update your .tfvars with any new variables
cd terraform/aws-ecs
terraform plan
terraform apply
```

## Major Features

### Pre-Release End-to-End Smoke Test

A new self-reporting, self-cleaning end-to-end smoke test that exercises the surface a release must not break, meant to be run against a live gateway right before cutting a release. It runs eight tests: registry health and configuration, the built-in `airegistry-tools` MCP server health plus its `intelligent_tool_finder` search tool, full CRUD lifecycles for servers, agents, and skills, cross-entity semantic search, security scanning, and one real external MCP server (the AWS knowledge base) reached end to end through the gateway proxy.

Key characteristics:

- Each test creates uniquely-named entities and tears them down even on failure, so the suite is safe to run repeatedly against a shared environment.
- The runner prints a pass/fail table and exits non-zero if any test fails, so it can gate a release from CI or a runbook.
- The external AWS test is marked SKIPPED (not FAILED) when the remote endpoint is unreachable, so a third-party outage never blocks a release.
- The `/release-notes` skill now confirms this smoke test was run and passed (or offers to run it) before any release-notes work begins.

Run it with:

```bash
uv run python tests/e2e_release_test.py --token-file .token --registry-url http://localhost
```

## Bug Fixes

- Fixed a 500 error on `POST /api/servers/toggle` (and any server re-index) when a server updated via PUT carried a null `tool_list` or `tags`. The search indexer used `dict.get(key, default)`, which returns `None` when the key is present-but-null, causing `TypeError: 'NoneType' object is not iterable` while building the embedding text. The register-then-edit-then-toggle flow now works reliably.
- Fixed the `RegistryClient` `ServerDetailResponse` model rejecting the null `num_tools` / `tool_list` a freshly-registered server returns before its tools are discovered.
- Fixed the `RegistryClient` `SecurityScanResult` model to match the actual `GET /api/servers/{path}/security-scan` response shape (top-level summary fields with the per-analyzer findings nested under `raw_output`), so scan results parse without error.
